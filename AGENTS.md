# RunThePaper Agent Guide

## Project Positioning

RunThePaper is the public output library for reproduced papers.

PRAgent/RRAgent is the harness we use to reproduce papers: agent workflows,
diagnostics, internal reference checks, source-material handling, and production
automation live there.

RunThePaper publishes the human-facing reproduction outputs after that work is
ready to share:

- reproduction notes;
- runnable reproduction code;
- generated data;
- generated figures;
- validation checks and scorecards;
- honest reproduction boundaries and lessons learned.

Do not treat RunThePaper as the harness repository. Do not move internal
PRAgent/RRAgent runtime code, checkpoints, prompts, private diagnostics, raw
paper sources, original figures, or source-derived comparison assets into this
repository.

RunThePaper is also not a model training-pack repository. Our service object is
the human reader: researchers, builders, students, and reviewers who want to
understand what was reproduced, how it was reproduced, what code can run, and
where the reproduction boundary is. If model companies later want to consume the
work for training, they can build their own training packs from the public
materials.

## Core Case Rule

Each public case should make five things easy to inspect:

1. what the paper claims;
2. what we reproduced;
3. how to run the reproduction code;
4. which data, figures, and checks were generated by us;
5. what remains uncertain, approximate, unavailable, or not redistributed.

Never hide uncertainty. Separate scientific agreement from visual similarity,
generated data from reference data, and feature-level reproduction from
author-data-level reproduction.
